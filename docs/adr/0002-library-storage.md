# ADR-0002：原文件不可变，支持托管副本与链接模式

- 状态：Accepted
- 日期：2026-07-19

## 背景

自动整理很容易被误解为自动重命名和移动原文件。网络盘、同步盘、损坏导入和错误元数据匹配都可能造成不可逆的数据问题。

## 决策

- 应用从不静默覆盖用户原文件。
- 默认导入模式为 `managed-copy`：复制到应用 library root，校验哈希成功后登记；原来源文件是否清理由用户显式决定。
- 首版只实现 `managed-copy`。对已有大型书库的 `linked` 模式后置：保持原目录，只建索引；文件丢失时标记 missing，不删除历史和阅读记录。
- 元数据和分类默认写数据库；写回 EPUB/PDF 或重命名/重组目录必须是独立、可预览的显式操作。
- 所有物理操作先形成计划，逐项记录结果，失败可重试。

## 理由

- 把自动分类与物理整理解耦，可显著降低误分类造成的数据损失。
- 内容哈希比路径更适合识别重复与移动。
- 两种模式同时覆盖“希望应用接管”和“已有目录不想动”两类用户。

## 备选方案

### 只索引、不复制

磁盘占用最少，但源路径变动、网络盘暂时离线和权限变化会直接破坏阅读体验。

### 自动移动并按作者/书名重命名

表面整洁，但最依赖元数据正确性，也最难安全回滚，不适合作为默认行为。

## 后果

- 托管模式会增加磁盘占用。
- 需要清晰展示文件所有权、来源路径和缺失状态。
- 备份应覆盖 SQLite、托管文件和应用配置；链接文件仍由用户自己的备份负责。
