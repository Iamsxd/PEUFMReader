# ADR-0003：自动分类采用分层、可解释、可撤销的管线

- 状态：Accepted
- 日期：2026-07-19

## 背景

“自动分类”可能指文件夹分组、外部元数据 subjects 映射、正文题材识别或 AI 标签。外部来源会冲突，模型会误判，用户分类体系也会持续变化。

## 决策

分类按以下顺序运行：用户锁定/规则 → 内嵌元数据 → ISBN 精确外部匹配 → 多字段模糊匹配 → 正文规则 → 可选模型建议。

每个候选保存来源、原值、归一化值、置信度、理由和决策状态。高置信度可自动采用但仍可撤销；中置信度进入待整理；低置信度不强行分类。首版不依赖 LLM。

用户修正会生成明确规则或训练样本，但系统不能未经提示把一次修正泛化成全库变更。批量重分类先支持预览/dry-run。

## 理由

- 电子书自带结构化元数据，确定性信息应先于概率模型。
- 受控分类便于筛选和统计，自由标签保留个性表达，两者不能混成一个无限标签池。
- 保存候选和决策历史后，可以替换提供者、调整阈值和重新分类，而不丢失人工劳动。

## 备选方案

### 只使用外部元数据分类

实现快，但分类词表混乱、中文覆盖不稳定，无法解释服务冲突。

### LLM-first

演示效果强，但带来成本、隐私、离线、输出稳定性和大规模复跑问题。它可以成为后续 adapter，不能成为数据模型前提。

### 仅按目录/文件名分类

适合已经整理好的书库，但无法处理散乱导入，也会把用户当前目录结构永久固化成领域模型。

## 后果

- 数据模型和 UI 比“直接写 category_id”复杂。
- 必须维护分类词表、映射规则和决策审计。
- 首版能用规则与元数据形成可靠闭环，后续再用真实人工决策评估是否值得加入模型。
