# M2 阅读、导入与运维闭环验证

日期：2026-07-20  
目标环境：Windows Docker Desktop（与 Unraid 使用同一 Compose 和镜像）  
目标规模：10 个并发用户、3000 本书

## 本阶段范围

- PDF/EPUB 目录、书内搜索、阅读状态切换。
- EPUB 按 CFI、章节路径、章节序号、整体进度逐级恢复。
- 多文件选择与拖放上传，2 路并发、逐文件进度与错误状态。
- 自动导入收件箱，文件稳定检测、持久化任务、失败隔离、人工重试。
- 后台任务百分比与阶段信息，服务重启后租约恢复。
- 登录失败限流、可信代理 IP、登录及管理员写操作审计。
- 书库一致性检查、停写快照备份和显式确认恢复。

## 3000 本书 / 10 并发性能基线

方法：在独立临时数据库中执行 `scripts/performance-seed.sql`，生成 3000 本 EPUB 书目、50 位作者和分类记录；使用 `scripts/performance-smoke.mjs` 对每个接口发起 200 次请求、并发度 10。测试结束后删除临时容器和数据库，不写入正式书库。

| 接口 | 吞吐（请求/秒） | P50 | P95 | 最大值 | 失败 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 首页聚合 | 130.3 | 53.3 ms | 135.9 ms | 613.9 ms | 0 |
| 最新书库 | 595.0 | 14.8 ms | 26.9 ms | 39.0 ms | 0 |
| 标题搜索 | 293.0 | 32.0 ms | 45.7 ms | 59.2 ms | 0 |
| 固定分类 | 251.9 | 38.4 ms | 49.6 ms | 70.0 ms | 0 |

当前局域网目标为常用列表接口 P95 小于 250 ms，结果达标。该测试衡量 API 和 PostgreSQL 查询，不包含真实 NAS 磁盘速度、反向代理 TLS、PDF 下载、OCR 或云端 AI 延迟。

## 上线前人工检查

1. 用至少一本目录复杂的 EPUB 和一本带书签的 PDF 检查目录定位、搜索结果跳转和刷新恢复。
2. 用扫描 PDF 验证 OCR 语言包、任务进度和 CPU 温度；大型 OCR 建议在非阅读高峰运行。
3. 在反向代理后启用 `COOKIE_SECURE=true`，并只把 `TRUSTED_PROXY_CIDR` 设置为实际代理网段。
4. 将 `scripts/backup.sh` 加入 Unraid 定时任务，并把 `${PEUFM_BACKUP_ROOT}` 再同步到另一台设备或离线介质。
5. 在副本环境执行一次 `scripts/restore.sh 快照名 --yes`，验证恢复时间和可用空间。
