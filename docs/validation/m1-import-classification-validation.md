# M1 导入与分类验证记录

> 日期：2026-07-19
> 结论：浏览器上传的 M1 导入、元数据、固定分类和管理员整理闭环已经打通。

## 已实现

- `002_catalog_and_classification.sql` 可从 M0 数据模型增量迁移，不要求清库。
- EPUB OPF：书名、作者、语言、出版年、ISBN、出版社、题材、简介和声明封面。
- PDF Info：书名、作者、创建年份、Subject 和 Keywords；没有可靠值时回退文件名并降低置信度。
- 作者实体与版本关联；书库支持作者、年份和已接受题材分组。
- 19 个固定题材；中英文规则分类最多生成三个带理由和置信度的决策。
- 高置信度内嵌题材可自动采用；书名或简介规则只进入待整理。
- 管理员可以查看所有元数据候选、来源和置信度，编辑规范值并确认固定题材。
- 任意书籍均可再次打开整理表单，因此自动采用的分类也可撤销或替换。
- 可选 Ollama structured-output 适配器和 OpenAI-compatible 适配器；返回值经过固定分类白名单验证，置信度上限为 0.89，只保存为建议。
- 导入任务记录成功、重复、警告和失败；来源文件不会被删除。
