# PEUFMReader 领域术语表

> 状态：Draft。用于避免产品、数据库和 UI 对同一概念使用不同名字。

| 中文术语 | 英文/代码名 | 定义 | 不等于 |
|---|---|---|---|
| 作品 | Work | 抽象的创作内容，不绑定具体出版社、语言或文件 | 文件、版本 |
| 版本 | Edition | 作品的一个出版版本，通常由语言、出版社、出版年、ISBN 区分 | EPUB/PDF 格式 |
| 书籍文件 | BookFile | 磁盘上的一个实际文件，包含格式、路径、哈希和解析状态 | 作品本身 |
| 受控分类 | Category | 由产品/用户维护的有限层级分类，可做统计和导航 | 自由标签、文件夹 |
| 自由标签 | Tag | 用户可任意添加的多值描述 | 唯一主分类 |
| 书架 | Shelf | 用户显式维护的一组书 | 分类树 |
| 智能书架 | SmartShelf | 保存的查询规则，结果随元数据变化自动更新 | 复制出来的书籍集合 |
| 元数据候选 | MetadataCandidate | 从文件或外部服务取得、尚待合并的字段建议，带来源和分数 | 已确认元数据 |
| 分类建议 | ClassificationSuggestion | 分类器对 Category/Tag 的建议 | 最终决策 |
| 分类决策 | ClassificationDecision | 系统或用户接受/拒绝某项分类建议的可审计记录 | 不可逆写入 |
| 导入收件箱 | ImportInbox | 新文件完成解析后、尚未通过低置信度复核的工作区 | 系统文件夹一定会被移动 |
| 托管副本 | ManagedCopy | 由应用复制到 library root 并负责生命周期的文件 | 原来源文件 |
| 链接文件 | LinkedFile | 保持在用户原路径、应用只索引和读取的文件 | 应用拥有的文件 |
| 文件身份 | FileIdentity | 以内容哈希为主，辅以大小、格式的去重身份 | 可变化的路径 |
| 阅读位置 | ReadingPosition | 格式相关的稳定锚点；EPUB 使用 CFI/href，PDF 使用页码/页内位置 | 单一百分比 |
| 阅读状态 | ReadingState | 某阅读者对某文件的最新进度、状态和更新时间快照 | 历史记录 |
| 阅读会话 | ReadingSession | 一段有开始、结束、有效时长和起止位置的阅读事件 | 应用窗口打开时间 |
| 批注 | Annotation | 绑定到内容锚点的高亮、笔记或书签 | 纯书籍元数据 |
| 用户 | User | 可以登录 Web 应用的账号，是阅读数据和权限的主体 | 共享书籍元数据 |
| 角色 | Role | 用户权限集合；MVP 使用 admin/reader | 单本书的分类 |
| 阅读者档案 | ReaderProfile | 某用户的阅读偏好、统计设置和显示配置 | 登录凭据本身 |
| 元数据锁 | MetadataLock | 用户要求某字段不再被自动覆盖的标记 | 禁止用户编辑 |
| 提供者 | MetadataProvider | 外部或本地元数据来源的可替换适配器 | 领域模型本身 |
| 后台任务 | Job | 可持久化、可恢复、可重试的导入/解析/索引工作单元 | 单次 UI 请求 |

## 建议统一的阅读状态

- `unread`：未开始。
- `reading`：已开始且未完成。
- `finished`：已完成。
- `paused`：主动搁置。
- `abandoned`：不打算继续，但保留历史。

“重读”不是覆盖 `finished`，而是开启新的阅读周期/会话集合，并保留历史完成记录。

## 待定术语

- 产品 UI 用“分类”还是“类别”。代码统一使用 `Category`。
- `Edition` 中文显示为“版本”还是“版次”。
- 用户面对的导入工作区叫“待整理”“收件箱”还是“新书待确认”。
